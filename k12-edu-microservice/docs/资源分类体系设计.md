## 七、数据库字典表设计

### 7.1 表结构

```sql
CREATE TABLE `dict` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `type` VARCHAR(50) NOT NULL COMMENT '类型：resource_type/textbook_version/exam_type/subject/grade_level/class_type',
  `code` VARCHAR(50) NOT NULL COMMENT '编码',
  `name` VARCHAR(100) NOT NULL COMMENT '名称',
  `sort` INT DEFAULT 0 COMMENT '排序',
  `description` VARCHAR(500) DEFAULT NULL COMMENT '描述',
  `deleted` TINYINT DEFAULT 0 COMMENT '删除标记',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_type_code` (`type`, `code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='字典表';
```

### 7.2 初始化脚本

执行 `sql/init_dict_data.sql` 初始化字典数据。

## 九、API 接口设计

### 9.1 获取字典列表

```
GET /api/dict/list
GET /api/dict/list?type=resource_type
```

响应：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "resource_type": [
      { "code": "paper_handout", "name": "讲义/练习册", "sort": 1 }
    ]
  }
}
```

### 9.2 资源筛选查询

```
GET /api/resource/list
```

参数：
- `gradeLevel`: 学段 (primary_1, junior_7, senior_10)
- `subject`: 学科 (chinese, math, english)
- `resourceType`: 资源类型 (paper_handout, media_micro)
- `textbookVersion`: 教材版本 (pep, bsd, oxford_discover)
- `examType`: 备考类型 (sync_term, entrance_zhongkao)
- `keyword`: 关键词搜索
- `sortBy`: 排序字段 (download_count, create_time, score)
