# 资源浏览页一周冲刺排班表（前端A / 前端B / 后端 / 测试）

> - `资源浏览页整体优化实施方案.md`

---

## 0. 冲刺范围（本周必须完成）

### 必达（P0）

1. 文案与编码问题修复（乱码、资料蓝->资料篮）
2. 生产页禁用 demo/mock 列表展示
3. 详情页禁用 mock 数据回退
4. 路由遗留 demo 入口处理
5. 真实数据跳转链路回归（列表 -> 详情）

### 争取（P1 核心）

1. SubjectDetailPage 按布局拆分第一版
2. 工具条统一（单份/成套/排序/批量）
3. 栏目分组 + 更多交互第一版

---

## 1. 角色分工说明

- **前端A（主程）**：主页面架构拆分、复杂状态链路
- **前端B（功能）**：文案修复、工具条统一、列表与详情交互
- **后端**：接口契约确认、真实数据保障、SQL体检与修复
- **测试**：回归用例设计、冒烟+全链路验证、上线验收

## 4. 验收口径（冲刺结束）

- [ ] 页面无乱码、术语统一“资料篮”
- [ ] 生产路径无 demo/mock 列表展示
- [ ] 列表点击仅走真实详情（`/resource/{id}`）
- [ ] 工具条交互在三种布局一致
- [ ] 拆分后主页面可维护性提升（结构清晰）
- [ ] SQL 体检关键异常为 0（或在阈值内）

---

## 5. 风险预案（本周重点）

### 风险 A：拆分导致行为回归

- **预案**：拆分仅搬迁不改逻辑；每次拆分后立即冒烟。

### 风险 B：真实数据不足导致空态增多

- **预案**：空态文案清晰 + 上传引导；后端补数并跑体检脚本。

### 风险 C：接口字段临时变更

- **预案**：契约文档冻结，新增字段只加不改；破坏性变更需评审。
