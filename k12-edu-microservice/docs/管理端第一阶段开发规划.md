# 管理端第一阶段开发规划

> 目标：后台独立进入、独立布局、独立鉴权  
> 原则：先搭入口与权限骨架，业务页面做空壳  
> 状态：**已实现（第一阶段）**

---

## 一、目标与边界

### 要做

- 后台独立路径 `/admin/*`
- 独立登录、布局、路由守卫
- 后端 `/api/admin/auth/me`、`/menus`、`/permissions`
- 7 个空壳页面 + 动态 Sidebar
- 最小 RBAC 表 + admin 鉴权拦截

### 不做（第二阶段）

- 资源 CRUD、审核流真实联调
- 分类目录 / 首页配置业务编辑
- 细粒度按钮权限 UI
- 独立 `k12-admin` 微服务

## 五、后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/auth/me` | 当前管理员信息 + roles + permissions |
| GET | `/api/admin/menus` | 动态菜单树 |
| GET | `/api/admin/permissions` | 权限 code 列表 |

鉴权：`X-User-Role=admin`（网关 JWT 透传），否则 401/403。

已有管理 API（需 admin）：

- `GET /api/admin/resource/**` → k12-resource
- `GET /api/admin/users/**` → k12-resource → k12-auth
- `GET /api/auth/users/**` → k12-auth（已加 admin 拦截）

## 六、网关路由

```yaml
- id: k12-admin-auth
  uri: http://localhost:8081
  predicates:
    - Path=/api/admin/auth/**,/api/admin/menus,/api/admin/permissions

- id: k12-admin-resource
  uri: http://localhost:8082
  predicates:
    - Path=/api/admin/**
```

## 七、前端守卫流程

```
进入 /admin/*
  → /admin/login|403 → 放行
  → 无 token → /admin/login?redirect=
  → bootstrap: me + menus + permissions
  → role !== admin → /admin/403
  → 路由 permission 不满足 → /admin/403
  → 放行
```

401 时：若在 `/admin` 路径下跳 `/admin/login`，否则跳 `/login`。

## 八、本地验证

### 1. 执行 SQL + 重启服务

```bash
# SQL
source sql/49_admin_rbac_baseline.sql
source sql/50_auth_admin_user_patch.sql

# 重启 k12-auth (8081)、k12-gateway (9001)
```

### 2. 管理端 API

```http
POST http://localhost:9001/api/auth/login
Content-Type: application/json

{"username":"admin","password":"admin123"}

GET http://localhost:9001/api/admin/auth/me
Authorization: Bearer <token>
```

### 3. 前端

```bash
cd k12-edu-platform
npm run dev
```

访问 `http://localhost:5173/admin/login`，使用 `admin / admin123` 登录。

## 九、验收标准

| # | 验收项 |
|---|--------|
| 1 | 未登录访问 `/admin` → 跳转 `/admin/login` |
| 2 | 教师账号登录后访问 `/admin` → 403 |
| 3 | admin 登录 → Dashboard 空壳 + Sidebar 7 项 |
| 4 | 刷新 `/admin/users` 保持登录 |
| 5 | 网关 9001 可调 `/api/admin/menus` |
| 6 | 非 admin 调 admin API → 403 |
