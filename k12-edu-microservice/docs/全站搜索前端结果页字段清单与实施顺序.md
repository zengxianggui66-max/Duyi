# 全站搜索：前端结果页字段清单与实施顺序

## 1. 目标

- 为顶部搜索提供“站内百度式”检索体验（仅检索本系统资源）。
- 统一前后端字段，避免联调阶段反复改字段名。
- 明确开发顺序（先可用、再增强、最后调优），支持小步快跑上线。

---

## 2. 前端结果页字段清单（对齐标准）

> 页面路由建议：`/search/result?q=xxx`

### 2.1 页面 Query 参数（URL）

| 字段 | 类型 | 必填 | 说明 | 示例 |
|---|---|---:|---|---|
| `q` | string | 是 | 搜索关键词 | `语文 教案` |
| `page` | number | 否 | 页码，默认 1 | `1` |
| `size` | number | 否 | 每页条数，默认 20 | `20` |
| `stage` | string | 否 | 学段筛选 | `primary` |
| `channel` | string | 否 | 频道筛选 | `competition` |
| `type` | string | 否 | 资源类型筛选 | `课件` |
| `sort` | string | 否 | 排序：`score/newest/download` | `score` |

### 2.2 搜索结果页主数据

#### `SearchAllResultVO`

| 字段 | 类型 | 说明 |
|---|---|---|
| `records` | `SearchResultItemVO[]` | 当前页结果 |
| `total` | number | 总条数 |
| `page` | number | 当前页 |
| `size` | number | 每页条数 |
| `pages` | number | 总页数 |
| `facets` | `SearchFacetsVO` | 筛选聚合统计 |
| `costMs` | number | 服务端耗时（可选） |

#### `SearchResultItemVO`（结果卡片字段）

| 字段 | 类型 | 必填 | 前端用途 |
|---|---|---:|---|
| `docId` | string | 是 | 唯一主键（`type:id`） |
| `resourceId` | number | 是 | 跳转详情使用 |
| `resourceType` | string | 是 | 数据源类型（`primary_chinese`/`topic`/`competition`...） |
| `title` | string | 是 | 结果标题 |
| `summary` | string | 否 | 摘要文本 |
| `titleHighlight` | string | 否 | 标题高亮（HTML） |
| `summaryHighlight` | string | 否 | 摘要高亮（HTML） |
| `stageKey` | string | 否 | 学段（`preschool/primary/senior/art/dance`） |
| `stageName` | string | 否 | 学段中文名 |
| `channelKey` | string | 否 | 频道（`class_meeting/career/culture/competition/topic`） |
| `channelName` | string | 否 | 频道中文名 |
| `subject` | string | 否 | 学科 |
| `gradeName` | string | 否 | 年级 |
| `teachingType` | string | 否 | 课件/教案/练习等 |
| `fileExt` | string | 否 | 文件扩展名 |
| `downloadCount` | number | 否 | 下载量 |
| `viewCount` | number | 否 | 浏览量 |
| `uploadTime` | string | 否 | 上传时间（ISO） |
| `score` | number | 否 | 综合得分（调试/灰度） |
| `detailRoute` | string | 是 | 前端直跳路由（如 `/resource/123?from=subject`） |
| `coverUrl` | string | 否 | 列表缩略图 |
| `badges` | string[] | 否 | 标签（如 `旗舰专页`、`精品`） |

### 2.3 Facets（筛选统计）

#### `SearchFacetsVO`

| 字段 | 类型 | 说明 |
|---|---|---|
| `stages` | `FacetBucket[]` | 学段聚合 |
| `channels` | `FacetBucket[]` | 频道聚合 |
| `types` | `FacetBucket[]` | 类型聚合 |

#### `FacetBucket`

| 字段 | 类型 | 说明 |
|---|---|---|
| `key` | string | 聚合 key |
| `name` | string | 展示名称 |
| `count` | number | 结果数 |

### 2.4 联想建议（搜索框下拉）

#### `SearchSuggestItemVO`

| 字段 | 类型 | 说明 |
|---|---|---|
| `text` | string | 建议词 |
| `kind` | string | `keyword/history/hot/title/channel` |
| `resourceType` | string | 可选，若是资源建议 |
| `resourceId` | number | 可选，若可直跳 |
| `detailRoute` | string | 可选，直跳路由 |
| `hitCount` | number | 可选，热度 |
| `highlight` | string | 可选，仅 `title/channel` 高亮文本（含 `<em>`） |

### 2.5 直跳判定（命中即跳）

#### `SearchRedirectVO`

| 字段 | 类型 | 说明 |
|---|---|---|
| `directHit` | boolean | 是否直接跳转 |
| `confidence` | number | 置信度（0~1） |
| `target` | `SearchResultItemVO \| null` | 直跳目标 |
| `reason` | string | 判定原因（可选） |

## 4. 执行顺序计划（开发先后）

## 阶段 P0（先可用）

1. 后端新增 `/search/all`（统一返回 `SearchAllResultVO`）
2. 前端新增搜索结果页 `SearchResultPage`（读取 URL 参数）
3. 顶部搜索框回车/按钮 -> 跳结果页（先不做直跳）
4. 接入基础筛选与分页

**验收**：输入关键词可稳定返回站内资源并跳转详情。

## 阶段 P1（体验增强）

1. 后端新增 `/search/suggest`
2. 前端搜索框增加联想下拉（防抖 + 键盘选择）
3. 后端新增 `/search/redirect`
4. 前端接入“命中即跳转”

**验收**：常见精确词可直接跳详情，多结果进入结果页。

## 阶段 P2（质量优化）

1. 高亮质量优化（标题+摘要）
2. 相关性权重调优（标题>标签>摘要）
3. 同义词、停用词、零结果推荐
4. 统计复盘（点击率、零结果率、直跳准确率）

**验收**：Top3 命中率提升、零结果率下降。

## 6. 回归清单（可直接执行）

> 目标：覆盖本轮 `suggest` 与 `AppHeader` 的关键路径，重点验证「分组/截断/追加词/键盘/高亮」。

### 6.1 后端接口回归（`GET /api/search/suggest`）

#### Case A：空查询（历史 + 热门）

- 请求：`q=""`，`limit=12`，`userId=<登录用户>`
- 预期：
  - 返回总数 `<=12`
  - 先 `history`（最多 5），再 `hot`（最多 5，按热度）
  - 不出现 `title/channel`
  - `text` 去重（大小写无关）

示例：

```bash
curl -G "http://localhost:8080/api/search/suggest" \
  --data-urlencode "q=" \
  --data-urlencode "limit=12" \
  --data-urlencode "userId=10001"
```

#### Case B：非空查询（title + channel + hot 补齐）

- 请求：`q="小学 教案"`，`limit=12`
- 预期：
  - `title` 最多 5（按“前缀 > 包含 > 热度/得分”）
  - `channel` 最多 3（主题班会/生涯规划/传统文化/竞赛专区/专题资源词典匹配）
  - `hot` 补齐至总数上限
  - 总数 `<=12`
  - `title/channel` 可返回 `highlight`（含 `<em>`）

#### Case C：limit 边界

- 请求：`limit=1 / 5 / 12 / 50`
- 预期：
  - 最终返回条数不超过 `min(limit,12)`
  - 不因大 limit 超过 12

#### Case D：停用词 + 同义词

- 输入词建议：
  - `q="的 了 资源"`（停用词）
  - `q="教学设计"`（同义词链路应命中教案类资源）
  - `q="PPT"`（同义词链路应命中课件类资源）
- 预期：
  - 停用词不导致异常噪音结果
  - 同义词查询仍能产出合理 `title` 建议

### 6.2 前端交互回归（`AppHeader`）

#### Case E：分组渲染与分组标题

- 操作：输入 `小学`，等待 220ms
- 预期：
  - 显示建议面板
  - 分组标题按语义显示：`资源直达 / 频道推荐 / 历史搜索 / 热门搜索`
  - 每组最多 5，总数最多 12

#### Case F：键盘导航

- 操作：输入关键词后按 `↓/↑`
- 预期：
  - 在 `flatSuggestions` 中循环移动
  - 高亮项与实际触发项一致
- 操作：按 `Esc`
- 预期：面板关闭
- 操作：按 `Enter`
- 预期：
  - 若当前项有 `detailRoute`：直跳
  - 否则：追加 token（不立即跳转）

#### Case G：多选择语义（连续追加）

- 操作：
  1. 点 `小学`
  2. 点 `语文`
  3. 点 `教案`
- 预期：
  - 输入框为：`小学 语文 教案`
  - 不自动跳转
  - 再按 Enter 才执行搜索

#### Case H：追加去重

- 操作：重复点击同一建议词（如 `教案`）2 次
- 预期：
  - 输入框不重复追加同 token（大小写不敏感）

#### Case I：高亮策略

- 操作：输入 `banhui` 或 `班会`（按实际词典命中）
- 预期：
  - 仅 `title/channel` 项应用 `<em>` 高亮
  - `history/hot` 不做高亮替换

### 6.3 一次性验收口径（通过标准）

- `suggest` 任意输入都满足：**可用、稳定、上限受控（<=12）**
- 前端联想交互满足：**可键盘、可鼠标、可连续选词、不误跳转**
- 搜索执行行为满足：**仅 Enter/点击搜索按钮触发统一检索**

## 7. P0+ 已落地（扁平索引 + QueryParser + 合规）

### 7.1 数据库

执行迁移脚本：

```bash
mysql -u root -p xinketang < k12-edu-microservice/sql/38_resource_search_index_p0plus.sql
```

表名：`resource_search_index`（跨 primary/topic/competition/culture 统一检索）

### 7.2 后端能力

| 模块 | 说明 |
|---|---|
| `SearchIndexSyncService` | 全量/增量写入扁平索引 |
| `SearchIndexQueryService` | 单表检索 + SQL facets + 真实 total/分页 |
| `SearchQueryParser` | 从 query 提取学段/频道/类型/学科/版本/年级/栏目 |
| `SearchGuardService` | 敏感词拦截 + 内存限流 + 未登录隐藏 VIP |
| `SearchIndexBootstrapRunner` | 索引为空时启动自动 rebuild |

### 7.3 运维接口

```bash
# 全量重建索引
curl -X POST http://localhost:8082/api/search/admin/reindex
```

### 7.4 配置项（application.yml）

```yaml
k12:
  search:
    rate-limit-per-minute: 30
    index-auto-rebuild-on-empty: true
    hide-vip-for-anonymous: true
    sensitive-words: [...]
```

### 7.5 P0+ 回归补充

| Case | 操作 | 预期 |
|---|---|---|
| J | `q=小学 语文 教案` | parser 拆 filter + token，索引命中，facets 为 SQL 聚合 |
| K | 未登录搜索 | `vip_flag=1` 资源不可见（配置开启时） |
| L | 1 分钟内连续搜索 31 次 | 返回 `queryHint=搜索过于频繁` |
| M | `q` 含敏感词 | 返回空结果 + queryHint |
| N | 索引表为空启动服务 | 自动异步 rebuild |

### 7.6 增量 sync 钩子（上传/审核）

| 触发点 | 类/方法 | 行为 |
|---|---|---|
| 学段资源上传 | `PrimaryChineseResourceService.save` | 异步 upsert/delete |
| 草稿保存/提交 | `saveDraft` / `submitDraft` | 草稿从索引移除；发布后写入 |
| 目录挂载 | `ResourcePlacementService.syncPrimaryPlacement` | catalog_path 更新后 sync |
| 专题上传 | `TopicZoneService.createResource` | upsert topic 索引 |
| 竞赛上传 | `CompetitionZoneService.createResource` | upsert competition 索引 |
| 文化上传 | `CultureStudyService.createResource` | upsert culture 索引 |

实现类：`SearchIndexSyncHook` → `SearchIndexSyncService.sync*ByIdAsync`

规则：`status=1` 且未删除 → upsert；否则 → 从索引删除。
