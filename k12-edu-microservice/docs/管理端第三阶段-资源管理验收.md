## 二、功能验收（12 项）

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| 1 | 资源列表 | super_admin 打开 `/admin/resources` | 15 列齐全，默认不含回收站 |
| 2 | 筛选 | 学段+学科+状态 | 结果正确 |
| 3 | 详情编辑 | 进入 `/admin/resources/:id` 改标题保存 | 保存成功并刷新 |
| 4 | 下架 | 已发布资源点下架 | status=3，前台搜索不可见 |
| 5 | 上架 | 已下架点发布 | status=1，搜索索引 upsert |
| 6 | 推荐/置顶 | 已发布资源操作 | 字段更新 |
| 7 | 回收站 | 删除 | status=4，列表默认不显示 |
| 8 | 恢复 | 筛选回收站 → 恢复 | status=3（已下架） |
| 9 | 批量下架 | 勾选已发布 ≥2 条 | 全部成功或跳过提示 |
| 10 | content_admin | 登录资源管理 | 仅小学语数范围 |
| 11 | auditor | 直接访问 `/admin/resources` | 403 或无菜单 |
| 12 | 操作日志 | 系统设置 → 操作日志，模块填 `resource` | 可见 publish/offline/batch/delete/restore |

## 三、审核与 Dashboard 一致性

1. 控制台「待审核」数量 = 审核中心列表 total（同一主表、同一 status=0 口径）
2. 审核通过 → 待审数 -1，已发布 +1
3. 验证 SQL：

```sql
SELECT COUNT(*) FROM oss_primary_chinese_resource
WHERE is_deleted = 0 AND status = 0;
```

---

## 四、搜索索引联动

审核通过 / 发布 / 下架 / 回收 / 恢复 后，异步同步：

- `sys_search_document`（doc_id = `primary_chinese:{id}`）
- `resource_search_index`
- OpenSearch（若已配置）

验证：

```sql
SELECT doc_id, title, status FROM sys_search_document
WHERE doc_id = 'primary_chinese:{id}';
```

```http
GET /api/search/all?q={标题关键词}&page=1&size=10
```
