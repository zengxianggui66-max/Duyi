# K12 教育资源平台 — 微服务启动指南

## 前置条件

| 组件 | 要求 | 确认命令 |
|------|------|----------|
| **JDK** | 21+ | `java -version` |
| **Maven** | 3.8+ | `mvn -version` |
| **MySQL** | 8.0+ | `mysql -u root -p` |
| **Node.js** | 18+ | `node -v` |
| **Nacos** | 2.x | 已下载并解压 |

## 架构总览

```
前端 (:5173) → Gateway (:9000) → [Auth(:8081) / Resource(:8082) / Article(:8083)]
                                    [Lesson(:8084) / Exam(:8085) / Member(:8086)]
                                           ↕
                                     Nacos (:8848) + MySQL (:3306)
```

```bash

# mysql -u root -pzxg123456 xinketang < k12-edu-server/src/main/resources/sql/init.sql
```

### 三、配置 Run Configuration（启动配置）

IDEA 导入项目后会自动识别 Spring Boot 启动类，但需要手动创建 7 个 Run Configuration。

#### 创建方式：逐个添加

1. 打开 `Run → Edit Configurations...`（或右上角下拉框 → `Edit Configurations`）
2. 点击左上角 `+` 号 → 选择 `Spring Boot`
3. 按下表填写，每创建一个点 OK，然后继续创建下一个

| 配置名 | 模块 | 主启动类 | Active Profiles | 端口 |
|--------|------|---------|----------------|------|
| **AuthApplication** | k12-auth | `com.k12.auth.AuthApplication` | — | 8081 |
| **ResourceApplication** | k12-resource | `com.k12.resource.ResourceApplication` | — | 8082 |
| **ArticleApplication** | k12-article | `com.k12.article.ArticleApplication` | — | 8083 |
| **LessonApplication** | k12-lesson | `com.k12.lesson.LessonApplication` | — | 8084 |
| **ExamApplication** | k12-exam | `com.k12.exam.ExamApplication` | — | 8085 |
| **MemberApplication** | k12-member | `com.k12.member.MemberApplication` | — | 8086 |
| **GatewayApplication** | k12-gateway | `com.k12.gateway.GatewayApplication` | — | 9000 |

> 💡 **快捷创建技巧**：直接打开启动类文件（如 `AuthApplication.java`），点击 `main` 方法左侧的绿色 ▶ 按钮 → `Run 'AuthApplication'`，IDEA 会自动创建配置。其他 6 个同理。

#### 快捷验证方式：智能搜索创建

在每个启动类文件中，你会看到 `main` 方法旁边有绿色运行按钮：
- 点击绿色 ▶ → `Modify Run Configuration...` → 确认 Module 和主启动类正确即可

### 七、IDEA 中常见问题排查

| 问题 | 现象 | 解决方案 |
|------|------|---------|
| Maven 依赖下载失败 | 红色波浪线、import 报错 | 打开 Maven 面板（右侧边栏），点击 🔄 Reload All Maven Projects |
| JDK 版本不对 | 编译报错 "Unsupported class file major version" | `File → Project Structure → Project` 确认 SDK 为 21 |
| Lombok 不生效 | getter/setter 报红 | `Settings → Plugins` 搜索安装 **Lombok**，重启 IDEA；`Settings → Build → Compiler → Annotation Processors` 勾选 **Enable annotation processing** |
| 端口冲突 | 启动报错 "Port 8081 was already in use" | `Settings → Edit Configurations` 中修改对应服务的 `server.port`，或在命令行 `netstat -ano | findstr :8081` 查杀占用进程 |
| 服务找不到主启动类 | Run Configuration 灰色 | 确认 Module 选择正确（如 AuthApplication 对应 k12-auth 模块） |
| 热部署不生效 | 修改代码后需手动重启 | 已在 pom.xml 中排除了 DevTools，当前不支持热部署，需手动重启 |
| 启动类识别不了 | 没有绿色运行按钮 | 右键点击启动类文件 → `Mark as → Spring Boot Configuration Class` |

## 常见问题排查

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| Nacos 连接失败 | Nacos 未启动或命名空间不存在 | 确认 Nacos 已启动，检查命名空间 `k12-edu` 是否已创建 |
| 服务注册不上 Nacos | Nacos 配置错误 | 检查 `application.yml` 中 `spring.cloud.nacos.discovery` 配置 |
| 网关 503 Service Unavailable | 目标服务未启动 | 检查对应的微服务是否已启动并注册到 Nacos |
| 前端接口 404 | 代理未指向网关 | 确认 `vite.config.ts` 中 `proxy.target` 为 `http://localhost:9000` |
| 数据库连接失败 | MySQL 未启动或密码不对 | 检查 MySQL 服务状态，确认 root/zxg123456 |
| 端口被占用 | 上次未正常关闭 | 查找占用端口的进程：`netstat -ano | findstr :端口号`，然后 `taskkill /PID 进程号 /F` |

## 切换回单体后端模式

如需切回单体后端 `localhost:8080`，只需修改前端代理：

```ts
// vite.config.ts
proxy: {
  '/api': {
    target: 'http://localhost:8080',  // 改回 8080
    changeOrigin: true,
  },
},
```
