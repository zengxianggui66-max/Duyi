# 全站搜索功能开发方案

## 1. 目标定义

全站式搜索不是简单按标题 `LIKE` 查询资源，而是把系统内所有可访问内容统一纳入一个搜索入口：

- 学段资源：幼儿、小学、初中、高中、美术、舞蹈，各年级、学科、版本、册别、单元、栏目、资源类型。
- 特色频道：主题班会、生涯规划、传统文化、竞赛专区、专题资源及其频道资料。
- 备课专区：备课中心、智能备课、组卷、学案、教案、课件、同步资源等。
- 教育资讯：教育局通知、政策资讯、资讯栏目、帮助中心等文章类内容。
- 系统入口：频道页、栏目页、筛选结果页、资源详情页、资讯详情页、专题详情页。

用户输入“语文”时，不应只返回一堆标题，而应先理解这是一个学科意图，给出“小学语文 / 初中语文 / 高中语文”等联动下拉建议；用户输入“一年级语文课件”“主题班会”“教育局通知”“备课教案”时，应能跳到最相关的频道、筛选页或详情页。

## 3. 总体架构

建议采用“搜索文档索引表 + 业务原表 + 可选搜索引擎”的架构。

### 3.1 MVP 架构

适合中小数据量、先快速上线：

1. 后端建立统一搜索索引表 `sys_search_document`。
2. 每类内容发布、更新、下架时同步写入索引表。
3. 查询时用 MySQL FULLTEXT + 中文分词预处理 + 规则打分。
4. 前端继续使用现有 `/search/all`、`/search/suggest`、`/search/redirect`。

### 3.2 增强架构

适合资源量变大、搜索体验要求更高：

1. MySQL 保存业务主数据和索引快照。
2. Elasticsearch / OpenSearch 保存倒排索引。
3. 后端通过异步任务或 MQ 同步索引。
4. 搜索支持拼音、错别字、同义词、短语召回、高亮、聚合分面、个性化排序。

推荐路线：先做 MySQL 索引表 MVP，接口和数据结构按搜索引擎兼容设计，后续平滑迁移 ES/OpenSearch。

## 4. 搜索文档模型

新增统一索引表：

```sql
CREATE TABLE sys_search_document (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  doc_id VARCHAR(80) NOT NULL UNIQUE COMMENT '全局文档ID，如 resource:123、news:45、channel:topic',
  doc_type VARCHAR(40) NOT NULL COMMENT 'resource/news/channel/lesson/prep/feature/page',
  biz_id VARCHAR(64) NOT NULL COMMENT '业务ID或业务key',
  title VARCHAR(255) NOT NULL,
  subtitle VARCHAR(255) DEFAULT '',
  summary TEXT,
  content_text MEDIUMTEXT COMMENT '可搜索正文/简介/文件解析文本',
  keyword_text TEXT COMMENT '标签、别名、同义词、拼音词、运营关键词',
  stage_key VARCHAR(30) DEFAULT NULL,
  stage_name VARCHAR(30) DEFAULT NULL,
  subject_key VARCHAR(30) DEFAULT NULL,
  subject_name VARCHAR(30) DEFAULT NULL,
  grade_key VARCHAR(30) DEFAULT NULL,
  grade_name VARCHAR(30) DEFAULT NULL,
  channel_key VARCHAR(50) DEFAULT NULL,
  channel_name VARCHAR(50) DEFAULT NULL,
  module_key VARCHAR(50) DEFAULT NULL,
  module_name VARCHAR(50) DEFAULT NULL,
  resource_type_key VARCHAR(50) DEFAULT NULL,
  resource_type_name VARCHAR(50) DEFAULT NULL,
  route_path VARCHAR(500) NOT NULL COMMENT '前端跳转地址',
  cover_url VARCHAR(500) DEFAULT '',
  publish_time DATETIME DEFAULT NULL,
  view_count INT DEFAULT 0,
  download_count INT DEFAULT 0,
  collect_count INT DEFAULT 0,
  quality_score DECIMAL(8,2) DEFAULT 0 COMMENT '人工/审核/精品权重',
  status TINYINT DEFAULT 1 COMMENT '1=可搜 0=不可搜',
  is_deleted TINYINT DEFAULT 0,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  FULLTEXT KEY ft_search_text (title, subtitle, summary, content_text, keyword_text),
  INDEX idx_type_status (doc_type, status, is_deleted),
  INDEX idx_stage_subject (stage_key, subject_key),
  INDEX idx_channel_module (channel_key, module_key),
  INDEX idx_publish_time (publish_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='全站搜索统一索引文档';
```

如果使用 Elasticsearch/OpenSearch，字段基本保持一致，额外增加 `suggest_text`、`pinyin`、`completion`、`boost`、`permissions` 等字段。

## 5. 内容来源映射

### 5.1 学段资源

来源表建议以现有设计中的 `edu_resource` 为主，并 JOIN：

- `edu_stage`
- `edu_subject`
- `edu_edition`
- `edu_volume`
- `edu_unit`
- `edu_module`
- `edu_resource_type`
- `edu_resource_tag`
- `edu_resource_file`

索引标题示例：

- `小学语文 一年级上册 统编版 第一单元 课件`
- `初中数学 七年级下册 期中试卷`
- `高中语文 必修上册 教案`

跳转策略：

- 明确资源：`/resource/:id`
- 明确学科但不明确资源：`/subject/:stage/:subject/:version?`，query 携带 `module/type/volume/keyword`
- 明确年级栏目：跳到对应筛选页，而不是只展示搜索结果。

### 5.2 特色频道

来源建议分两类索引：

- 频道入口文档：主题班会、生涯规划、传统文化、竞赛专区、专题资源。
- 频道内资源文档：频道分类、专题资源、资料详情。

现有前端配置可复用：

- `src/constants/featureChannelRegistry.ts`
- `src/views/feature/*`
- `src/components/feature/*`

跳转策略：

- “主题班会” -> `/theme-class-meeting`
- “生涯规划” -> `/topic` 或当前配置中的 `/feature/shengya` 重定向链路
- “传统文化” -> `/culture`
- “竞赛专区” -> `/competition`
- “专题资源” -> `/topic`
- 明确资源详情 -> `/topic-zone/resource/:id` 或 `/competition-zone/resource/:id`

### 5.3 备课专区

索引内容：

- 备课中心入口：`/lesson`
- 智能备课：`/lesson/smart`
- 试卷/组卷：`/lesson/papers`、`/lesson/assemble`
- 资料篮：`/lesson/basket`
- 与备课相关的资源类型：教案、课件、学案、教学设计、讲义、公开课、说课稿。

当用户搜索“备课”“教案”“课件”“教学设计”时，优先展示备课专区入口和对应资源列表。

### 5.4 教育资讯

来源：

- `news` 相关接口和页面：`src/api/news.ts`、`src/store/news.ts`、`src/views/news/*`
- 教育局通知、政策资讯、资讯频道、帮助中心文章。

跳转策略：

- 明确资讯文章 -> `/news/:id`
- 明确资讯频道 -> `/news/channel/:key`
- 泛化“教育资讯”“教育局通知” -> `/news` 或 `/news/list?keyword=...`

## 6. 核心算法

### 6.1 中文分词与标准化

输入处理：

1. 去空格、全半角统一、大小写统一。
2. 中文分词：可用 IK Analyzer、jieba、HanLP、结巴 Java 版等。
3. 同义词扩展：
   - 语文 = 中文 = 国语
   - 数学 = 奥数
   - 备课 = 教案 = 教学设计 = 课件
   - 资讯 = 新闻 = 通知 = 政策
   - 主题班会 = 班会 = 德育
4. 年级归一：
   - 一年级 / 1年级 / 小一 -> `grade=一年级`
   - 初一 -> 七年级
   - 初二 -> 八年级
   - 初三 -> 九年级
   - 高一/高二/高三 -> 高中年级。
5. 学段归一：
   - 小学、初中、高中、幼儿、美术、舞蹈。

### 6.2 意图识别

把搜索词解析成结构化意图：

```json
{
  "raw": "小学语文一年级课件",
  "stage": "primary",
  "subject": "chinese",
  "grade": "一年级",
  "resourceType": "courseware",
  "channel": null,
  "module": "sync_prep",
  "keyword": ""
}
```

识别优先级：

1. 精确频道名：主题班会、生涯规划、传统文化、竞赛专区、专题资源。
2. 学段 + 学科 + 年级 + 类型组合。
3. 备课/考试/资讯等功能域。
4. 资源标题和正文模糊搜索。
5. 热词、历史、运营推荐兜底。

### 6.3 召回算法

建议多路召回后合并去重：

- 精确匹配：标题、频道名、资源类型、学科名。
- 分词匹配：标题、摘要、正文、标签。
- 维度匹配：stage、subject、grade、module、type。
- 同义词匹配：教案/教学设计/备课。
- 拼音匹配：后续增强，例如 `yuwen` -> 语文。
- 热门补充：无结果时推荐热搜、频道入口、相邻年级/学科。

### 6.4 排序算法

综合分建议：

```text
score =
  文本相关度 * 0.45
+ 意图匹配度 * 0.25
+ 资源质量分 * 0.12
+ 热度分 * 0.08
+ 新鲜度分 * 0.06
+ 点击反馈分 * 0.04
```

加权规则：

- 标题命中 > 标签命中 > 摘要命中 > 正文命中。
- “小学语文”这类维度词优先返回学科入口和筛选页。
- “教案/课件/试卷”这类类型词优先返回资源列表和资源详情。
- “主题班会/传统文化”这类频道词优先频道入口。
- 用户点过的结果可通过点击日志提高后续排序。

### 6.5 搜索直达

`/search/redirect` 应做高置信度判断：

- 完全命中频道名：直接跳频道。
- 完全命中唯一资源标题：直接跳详情。
- 命中“学段 + 学科”但有多个阶段：不直接跳，返回建议下拉。
- 命中“语文”这种泛词：返回小学语文、初中语文、高中语文联动菜单。

置信度示例：

- `confidence >= 0.90`：直接跳。
- `0.60 <= confidence < 0.90`：展示下拉建议。
- `< 0.60`：进入搜索结果页。

## 9. 数据库设计评估

现有 `docs/database-schema.sql` 里的 `edu_stage`、`edu_subject`、`edu_edition`、`edu_volume`、`edu_unit`、`edu_module`、`edu_resource_type`、`edu_resource` 方向是规范的，适合做搜索维度来源。

需要补强：

- 缺统一搜索索引表：新增 `sys_search_document`。
- 缺搜索词典：新增 `sys_search_synonym`、`sys_search_intent_rule`。
- 缺点击反馈：已有前端 `recordClick`，后端应落表 `sys_search_click_log`。
- 缺零结果分析：新增或复用搜索日志表记录 `hit_count=0`。
- 缺资讯/特色频道与资源统一索引模型：通过 `doc_type + route_path` 统一。
- `edu_resource` 当前是一资源一文件，长期建议拆 `edu_resource_file` 支持主文件、答案、解析、附件、音视频。

搜索相关补充表：

```sql
CREATE TABLE sys_search_synonym (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  word VARCHAR(80) NOT NULL,
  synonyms VARCHAR(500) NOT NULL COMMENT '逗号分隔或JSON',
  domain VARCHAR(40) DEFAULT 'global',
  status TINYINT DEFAULT 1,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE sys_search_intent_rule (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  pattern VARCHAR(200) NOT NULL COMMENT '关键词或正则',
  intent_type VARCHAR(40) NOT NULL COMMENT 'subject/channel/prep/news/resource',
  target_route VARCHAR(500) DEFAULT '',
  target_payload JSON,
  priority INT DEFAULT 0,
  status TINYINT DEFAULT 1
);

CREATE TABLE sys_search_query_log (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  keyword VARCHAR(200) NOT NULL,
  normalized_keyword VARCHAR(200) DEFAULT '',
  user_id BIGINT UNSIGNED DEFAULT NULL,
  hit_count INT DEFAULT 0,
  cost_ms INT DEFAULT 0,
  search_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_keyword (keyword),
  INDEX idx_search_time (search_time)
);

CREATE TABLE sys_search_click_log (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  keyword VARCHAR(200) NOT NULL,
  doc_id VARCHAR(80) DEFAULT '',
  doc_type VARCHAR(40) DEFAULT '',
  biz_id VARCHAR(64) DEFAULT '',
  user_id BIGINT UNSIGNED DEFAULT NULL,
  position INT DEFAULT NULL,
  click_type VARCHAR(30) DEFAULT 'result',
  route_path VARCHAR(500) DEFAULT '',
  click_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_keyword_time (keyword, click_time),
  INDEX idx_doc_id (doc_id)
);
```

4. 增加权限过滤、个性化排序、热门趋势。

## 12. 验收用例

- 搜索“语文”：下拉出现小学语文、初中语文、高中语文，可分别跳转。
- 搜索“小学语文一年级课件”：优先进入小学语文筛选结果或展示对应资源。
- 搜索“主题班会”：可直达主题班会频道。
- 搜索“生涯规划”：可直达生涯规划频道或专题页。
- 搜索“传统文化”：可直达传统文化频道。
- 搜索“竞赛专区”：可直达竞赛专区。
- 搜索“备课”：展示备课中心、智能备课、教案课件资源。
- 搜索“教育局通知”：展示教育资讯/通知类文章。
- 搜索不存在内容：展示相近推荐和热门入口，不出现空白死页。
- 点击任意搜索结果：记录点击日志，并跳转到正确业务页面。

## 13. 推荐优先级

第一优先级不是马上上 Elasticsearch，而是把“可搜索对象”统一成 `sys_search_document`。只要所有资源、频道、资讯、备课入口都有统一文档、统一 `route_path`、统一分面字段，前端当前搜索框就能真正变成全站入口。

后续搜索越强大，主要靠三件事：

1. 索引内容足够全。
2. 意图识别足够准。
3. 点击和零结果数据持续反哺排序。
