## P0：联调阻塞项

### 1. 网关路由（`k12-gateway`）

在 `k12-gateway/src/main/resources/application.yml` 中新增转发到 **8082** 的路由：

| 路由 ID | 路径前缀 |
|---------|----------|
| `k12-file` | `/api/file/**` |
| `k12-primary-chinese` | `/api/primary-chinese/**` |
| `k12-edu-resource` | `/api/edu-resource/**` |
| `k12-search` | `/api/search/**` |
| `k12-unit-doc` | `/api/unit-doc/**` |

原有 `/api/resource/**` 仍转发收藏、分享等子路径（如 `/api/resource/collect`）。

**网关白名单（匿名可读）补充：**

- `primary-chinese`：分页/列表/筛选项/单元名/单元树/成套/栏目统计等
- 数字 ID 详情：`/api/primary-chinese/{数字}` 及 `/{id}/view`、`/{id}/download`（正则放行，**不放行** `/save`）
- `edu-resource/**`、`/api/file/formats|check-format|preview|preview-info`、`/api/search/hot-keywords`

### 2. 小学语文资源字段对齐

**新增 `PrimaryChineseResourceVO`**（`k12-common`）

- 对外返回 `description`（映射库表 `remark`）
- 支持 `lessonName`、`allowPreview`
- 列表 / 分页 / 详情 / `save` 均返回 VO

**实体扩展**（`oss_primary_chinese_resource`）

| 字段 | 说明 |
|------|------|
| `lesson_name` | 课文名（目录叶子节点） |
| `allow_preview` | 是否允许预览：0/1 |

**保存时用户 ID**

- `UserIdResolver`：从 `X-User-Id`（网关透传）或 `Authorization: Bearer` 解析
- `JwtUserIdFilter`：直连 8082 时同样支持 JWT
- `POST /api/primary-chinese/save` 自动写入 `uploaderId`

### 3. 收藏 `resourceType`

避免 `primary_chinese` 与旧 `resource` 表 **主键 ID 冲突**。

**表结构**（`collection`）

- 新增 `resource_type`：`resource` | `primary_chinese`
- 唯一索引：`(user_id, resource_id, resource_type)`

**接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/resource/collect` | Body: `{ resourceId, resourceType? }` |
| DELETE | `/api/resource/collect/{resourceId}` | Query: `resourceType` |
| GET | `/api/resource/collect/check/{resourceId}` | Query: `resourceType` |

**前端**

- `collectApi` 增加 `resourceType` 参数
- 学科详情（`from=subject`）使用 `primary_chinese`

### P0 关键类

- `com.k12.common.dto.PrimaryChineseResourceVO`
- `com.k12.common.constant.ResourceTypeConstants`
- `com.k12.common.util.UserIdResolver`
- `com.k12.resource.config.JwtUserIdFilter`

## P1：体验增强

### 1. 草稿能力

**状态约定**（`ResourceStatusConstants`）

| status | 含义 |
|--------|------|
| `-1` | 草稿（仅上传者可见） |
| `0` | 待审核 |
| `1` | 已发布 |
| `2` | 审核不通过 |
| `3` | 下架 |

列表查询默认 `status >= 0`，草稿不出现在学科浏览列表。

**接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/primary-chinese/draft/save` | 新建/更新草稿（可无文件） |
| GET | `/api/primary-chinese/drafts` | 当前用户草稿列表（需登录） |
| GET | `/api/primary-chinese/draft/{id}` | 读取单条草稿 |
| POST | `/api/primary-chinese/draft/{id}/submit` | 提交审核（-1 → 0，需已上传文件） |

**前端**

- 上传页「保存草稿」已对接
- URL 支持 `?draftId=` 恢复草稿
- 正式提交：有 `draftId` 时先更新草稿再 `submitDraft`

### 2. 单元树 API

**接口**

```
GET /api/primary-chinese/unit-tree
  ?volumeKey=y1s2
  &gradeName=一年级下册
  &edition=统编版
  &subject=语文
```

**响应示例**

```json
[
  {
    "name": "第一单元·识字",
    "subUnits": ["春夏秋冬", "姓氏歌"]
  }
]
```

**数据来源**

1. 静态配置：`k12-edu-microservice/k12-resource/src/main/resources/catalog/unit-catalog.json`（与前端 `src/config/unitData.ts` 一致）
2. 合并库中 `unit_name` + `lesson_name`
3. 无静态数据时回退 DB 扁平单元名

**前端**：`useUnitDirectory` 改为调用 `primaryChineseApi.getUnitTree`。

### 3. 结构化教案 `lessonPlanJson`

- 库表字段：`lesson_plan_json`（TEXT）
- VO / 实体已支持
- 详情页 `resolveLessonPlanContent` 可解析 `lessonPlanJson` / `remark` / `description`

## P2：架构与业务能力

### 1. 成套资源 API

**接口**

```
GET /api/primary-chinese/suites
  ?stage=小学&subject=语文&gradeName=...&edition=...&module=...&unitName=...
```

服务端按 **资源类型** 聚合，返回 `ResourceSuiteVO` 列表（含 `items`）。

**前端**：学科页「找成套」改为 `primaryChineseApi.getSuites`，不再前端 `getList` 后分组。

### 2. OSS 元数据统一

**`OssMetadataHelper`**

| URL 类型 | oss_bucket | oss_object_key |
|----------|------------|----------------|
| 本地上传 `upload.base-url/...` | `local` | 相对路径，如 `2026/05/16/uuid.docx` |
| MinIO 公网 URL | `minio.bucketName` | URL 路径部分 |

- `save` / `saveDraft` 保存前自动填充
- `POST /api/file/upload` 额外返回 `ossBucket`、`ossObjectKey`

### 3. 分享能力增强

**表结构**：`share_record.resource_type`（`resource` | `primary_chinese`）

**链接规则**（`app.frontend-base-url`）

| resourceType | 分享 URL |
|--------------|----------|
| `primary_chinese` | `{frontend}/resource/{id}?from=subject` |
| `resource` | `{frontend}/resource/{id}` |

**接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/resource/share/url/{resourceId}` | Query: `resourceType` |
| POST | `/api/resource/share/record` | Body 含 `resourceType` |
| GET | `/api/resource/share/count/{resourceId}` | Query: `resourceType` |

**前端**：详情页打开分享弹窗调用 `getShareUrl`；复制链接时 `recordShare`。

### 4. 维度数据对齐

| 接口 | 说明 |
|------|------|
| `GET /api/primary-chinese/module-stats` | 按栏目统计资源数（对齐 edu-resource） |
| `GET /api/edu-resource/units` | 优先返回带 `subUnits` 的单元树；支持 `volumeKey`；无配置时回退原 DB 查询 |

## 数据库迁移

按顺序在 MySQL 中执行（按实际 schema 调整库名）。脚本位于 `k12-edu-microservice/k12-resource/src/main/resources/sql/`。

### P0 — `p0_migration.sql`

```sql
-- 收藏表 resource_type
ALTER TABLE `collection`
    ADD COLUMN `resource_type` VARCHAR(32) NOT NULL DEFAULT 'resource'
        COMMENT 'resource|primary_chinese' AFTER `resource_id`;

ALTER TABLE `collection` DROP INDEX IF EXISTS `uk_user_resource`;
ALTER TABLE `collection`
    ADD UNIQUE KEY `uk_user_resource_type` (`user_id`, `resource_id`, `resource_type`);

-- 小学语文资源表
ALTER TABLE `xinketang`.`oss_primary_chinese_resource`
    ADD COLUMN `lesson_name` VARCHAR(128) DEFAULT NULL COMMENT '课文名称' AFTER `unit_name`;

ALTER TABLE `xinketang`.`oss_primary_chinese_resource`
    ADD COLUMN `allow_preview` TINYINT NOT NULL DEFAULT 1 COMMENT '是否允许预览' AFTER `remark`;
```

> MySQL 8.0.12 以下不支持 `IF NOT EXISTS`，请去掉该语法手动执行。

### P1 — `p1_migration.sql`

```sql
ALTER TABLE `xinketang`.`oss_primary_chinese_resource`
    ADD COLUMN `lesson_plan_json` TEXT DEFAULT NULL COMMENT '结构化教案JSON' AFTER `allow_preview`;
```

### P2 — `p2_migration.sql`

```sql
ALTER TABLE `share_record`
    ADD COLUMN `resource_type` VARCHAR(32) NOT NULL DEFAULT 'resource'
        COMMENT 'resource|primary_chinese' AFTER `resource_id`;
```

**新建库初始化**：`init_resource_tables.sql` 中 `collection` 表已包含 `resource_type` 与联合唯一索引。

## 已知限制与后续建议

1. **双数据源**：浏览主走 `primary-chinese`，部分筛选项仍用 `edu-resource` 维度表，长期可考虑统一主表或同步脚本。
2. **成套资源**：当前为按类型**动态聚合**，非持久化资源包实体；若需「创建包、一键下载」需新增 `resource_suite` 表。
3. **审核后台**：`status=0` 待审核流程需管理端对接 `AdminResourceController`。
4. **docx 水印**：学科网源文件嵌入水印，mammoth 原样渲染，非系统添加。
5. **临时文件清理**：预览临时文件 TTL / 定时任务尚未实现。
